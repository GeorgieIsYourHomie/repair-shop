# User Stories for Repair Shop App

1. [ ] Replace Sticky Note System
Replace messy onboarding checklists with one centralized digital system.

2. [ ] Public Page
Provide a public-facing page with basic agency info and a login button.

3. [ ] Authentication
Add passwordless employee login (Magic Link OR GitHub provider).

4. [ ] Admin Dashboard
After admin login, show a real-time “Employees in Onboarding” list.

5. [ ] Employee Search/Filter
Allow admins to quickly search and filter employees.

6. [ ] Logout
Provide a logout option.

7. [ ] Login Requirement
Require users to log in at least once per week.

8. [ ] Disable Access
Allow admins to instantly disable employee access.

9. [ ] Employee Data Model
Employees have: ID, full address, phone, email, role, and notes.

10. [ ] Onboarding Step Data Model
Onboarding steps have: ID, title, description, status, created/updated dates.

11. [ ] Onboarding Step States
Steps are either OPEN or COMPLETED.

12. [ ] Step Assignment
Steps are assigned to specific employees.

13. [ ] Permissions
Users can have Employee or Admin permissions.

14. [ ] Admin Abilities
Admins can create & view employee records.

15. [ ] Template Management
Admins can create, edit, and assign onboarding templates.

16. [ ] Employee Abilities
Employees can only view and complete their own onboarding steps.

17. [ ] Admin Step Control
Admins can view, edit, or reset all onboarding steps.

18. [ ] Desktop-First UX
App must work best on desktop, but support tablets as well.

19. [ ] Light / Dark Mode
Use Shadcn UI theme system to provide light/dark mode.

20. [ ] Support Monitoring
Integrate Sentry for quick troubleshooting if anything breaks.
